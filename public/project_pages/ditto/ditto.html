DITeTriO is a project that aims to play the online stacker game TETR.IO while capturing the nuances of human playstyles (hence, DITTO + TETR.IO). Training was done on the Perlmutter supercomputer at NERSC. The project has 3 main components: data processing, model architecture, and evaluation.

Data processing

  • Uses the open source TetrioVirtualEnvironment to process replay files
  • Dataset of 1 month of top 100 players’ games
  • C# data preprocessor which converts replays from action sequences into sequences of game state and action pairs

Model architecture

  • CNN to extract features from 2d board
  • LSTM to extract time-series features from button presses over time
  • Combined features are put through linear layers to decide on probabilities for each button being pressed next

Evaluation

  • Board state visualizer made in PyGame to display input states and predicted button probabilities
  • Live communication with Tetrio Virtual Environment using a C# interface and inter-process communication to Python
  • Visualization of live state as the trained model interacts with the virtual environment

Results

  • The model has a difficult time making good stacking decisions due to the granularity of the action space compared to the overall goal
  • Generalization was difficult to achieve since the chosen representation for the pieces didn’t capture their spacial properties well
  • More work will be done to experiment with better state representations since the framework and pipeline for training and data processing are complete.
